## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input (wizard) → filesystem | Wizard answers become the first `config.toml`. Injected content crosses here. |
| user input (`config set`) → filesystem | CLI arg value written to `[routing].mode` etc. Injected content crosses here. |
| slash command (`/mode ...`) → filesystem | TUI-parsed command modifies config on disk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | Tampering | config.toml corruption on partial write | mitigate | Atomic write (tmp file + `os.replace`) in `write_initial_config` and `set_config_value` (Task 1 acceptance_criteria grep-verifies both). |
| T-01-14 | Information Disclosure | config.toml world-readable | mitigate | `os.chmod(path, 0o600)` after every atomic replace (Task 1 acceptance_criteria grep-verifies both). |
| T-01-15 | Tampering | Wizard/`config set` writes invalid mode | mitigate | Both writers reject `mode` values not in `{"local","online"}` (Task 1 <behavior>); subsequent `load_config()` re-validates via `Literal` (Plan 02). |
| T-01-16 | Injection | `config set` value containing TOML metacharacters (`"` or newline) | mitigate | `_serialise` in Task 1 raises `ValueError` on `"` or newline in string values — closed schema doesn't need them. |
| T-01-17 | Tampering | Malicious slash command from a compromised terminal paste | accept | Single-user tool; slash command is scoped to `/mode local`/`/mode online` only; unknown commands are silent no-ops. Acceptable. |
</threat_model>
