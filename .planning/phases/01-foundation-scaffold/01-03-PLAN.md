<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process → filesystem | mkdir + probe both cross into the OS. Permission modes and path escapes are the concerns. |
| user config → probe target | `probe.probe_host` from config decides where the TCP probe reaches out. A malicious config could redirect the probe. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Information Disclosure | World-readable workspace on shared Jetson | mitigate | `os.umask(0o077)` at CLI entry (Task 3); `test_umask_applied` verifies mode 0o700 (Task 1). |
| T-01-10 | Tampering | `data_dir` symlink attack | accept | Single-user local tool (RESEARCH.md Security Domain). Acceptable in v1.0; document that `data_dir` should not point outside the user's own tree. |
| T-01-11 | DoS / Info Disclosure | Probe host misconfigured to internal address | accept | User-controlled by design (D-05 exposes `probe.probe_host`). Single-user tool; user targets their own network. |
| T-01-12 | Tampering | TOCTOU race in mkdir | mitigate | `mkdir(exist_ok=True)` — no pre-existence check (anti-pattern check in Task 1 acceptance_criteria). |
</threat_model>
