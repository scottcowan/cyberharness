    - `atomic_write_json(path: Path, model: BaseModel) -> None` is `await asyncio.to_thread(_atomic_write_sync, path, model.model_dump_json(by_alias=True, indent=2))`
    - `_atomic_write_sync(path, data)` does: `os.open(tmp, O_WRONLY|O_CREAT|O_TRUNC, 0o600)` → `os.write(fd, data.encode('utf-8'))` → `os.fsync(fd)` → `os.close(fd)` → `os.replace(tmp, path)` where `tmp = path.with_suffix(path.suffix + '.tmp')`
    - `SessionStore(root: Path, workspace_id: str = "default")` creates `<root>/<workspace_id>/{active,archive}/` with mode 0o700
    - `SessionStore.create()` returns a `LoadedSession` — mints UUID, acquires filelock nonblocking on `<uuid>.lock`, writes initial JSON, returns dataclass `LoadedSession(session=Session, lock=FileLock, path=Path, save_lock=asyncio.Lock)`
    - `SessionStore.save(loaded: LoadedSession) -> None` acquires the per-session asyncio.Lock (in-process; Pitfall 1), sets `updated_at=datetime.now(timezone.utc)`, calls atomic_write_json
    - `SessionStore.load(path: Path) -> LoadedSession` acquires filelock nonblocking; on filelock.Timeout re-raises as `SessionLockedError`
    - Filelock library: `filelock.FileLock(str(lock_path))` per RESEARCH.md; called inside `asyncio.to_thread` for acquire
    - test_store.py GREEN cases: test_atomic_write_creates_0600_file, test_atomic_write_no_tmp_left, test_no_corrupt_on_kill (spawn subprocess that writes tmp then SIGKILL before rename — the prior version stays), test_filelock_blocks_second_instance (two FileLock instances on same path — second raises Timeout)
  </behavior>
  <action>
    Implement `store.py` with `_atomic_write_sync`, `atomic_write_json`, `LoadedSession` (dataclass with `session`, `lock`, `path`, `save_lock: asyncio.Lock`), `SessionLockedError` (raised when filelock cross-process acquire fails), and `SessionStore` class with `__init__`, `create`, `save`, `load`. Directory permissions: `active_dir.mkdir(parents=True, exist_ok=True); os.chmod(active_dir, 0o700)` (chmod is required — mkdir mode is masked by umask). Session JSON permissions: mode 0o600 comes from the `os.open` flags in `_atomic_write_sync` — do NOT rely on umask alone. Per-session `asyncio.Lock` MUST be created inside `create()` and stored on `LoadedSession` (Pitfall 1: serialise saves against auto-title race). The filelock MUST live for the whole active lifetime of the session — do NOT release in save(). Do NOT swallow filelock.Timeout in create() — a UUID collision on lock acquire is a bug, not an expected condition. For the crash-mid-write test: spawn a subprocess via `subprocess.Popen([sys.executable, "-c", "..."])` that opens the tmp fd, writes, then calls `os.kill(os.getpid(), signal.SIGKILL)` before rename — assert the original file's content is unchanged and parseable. Export `SessionStore`, `LoadedSession`, `SessionLockedError`, `atomic_write_json` from `session/__init__.py`. Update session/__init__.py __all__ to include these.
