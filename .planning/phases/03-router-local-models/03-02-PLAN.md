Implement `LocalModelClient` — the single-class HTTP consumer for both Ollama and LM Studio via their shared `/v1/chat/completions` OpenAI-compat endpoint. Delivers streaming token emission (SSE parsed via `response.aiter_lines()`), fire-and-forget warm-up with `keep_alive=-1`, Ollama-specific version advisory, `list_models()` for Phase 4 bench, and a single process-lifetime `httpx.AsyncClient` (D-05).

Purpose: This is the wire-level implementation that makes ROUT-03 (Ollama streaming + warm-up) and ROUT-04 (LM Studio via configurable base_url) real. The Router in 03-04 wraps this class; nothing else in Phase 3 owns the HTTP boundary.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| harness process → localhost Ollama/LM Studio | Local endpoints have no auth in v1.0 (bind localhost). Payload sanitisation still required to avoid triggering server-side parser bugs (Ollama #16932, TODO-001). |
| server SSE stream → harness parser | Untrusted content: malformed frames, `[DONE]` sentinel, embedded JSON. Tolerant parser required (Pitfall 1). |
| server streaming response → downstream consumer (eventually TUI in Phase 5) | Model output may contain terminal escape sequences; **Phase 3 does NOT sanitise content** — that is the TUI's rendering responsibility (Phase 5). Documented as accepted risk here so it isn't lost. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service | LocalModelClient — loop blocking | mitigate | All HTTP via `httpx.AsyncClient`; no sync `requests` / `socket` import. `timeout=httpx.Timeout(connect=10, read=None, write=10, pool=5)` bounds connect/write; streaming read intentionally uncapped so long generations don't spuriously abort (Pitfall 1, A6). |
| T-03-05 | Tampering | SSE frame parsing | mitigate | `aiter_lines()` handles TCP fragmentation; `try/except JSONDecodeError` on each frame; break on `[DONE]`; skip frames without `data: ` prefix (D-04, Pitfall 5). |
| T-03-06 | Denial of Service | Ollama cold start freezing UI | mitigate | Fire-and-forget `warm_up()` at startup with `keep_alive: -1` pins model in VRAM (D-06, Pitfall 4); wrapped in try/except so warm-up failure never crashes boot. |
| T-03-07 | Tampering / DoS | Ollama < 0.30.12 brace-detection tool-call bug | mitigate | `check_version()` warns operator; `to_wire()` allowlist + ToolRegistry `name`-param rejection (from 03-01) pre-empt the trigger conditions (D-11, TODO-001 §3). |
| T-03-08 | Information Disclosure | Model output rendered to terminal (XSS-equivalent via ANSI escapes) | accept | Phase 3 emits raw content chunks — the TUI (Phase 5) owns rendering safety. Recorded here so Phase 5 planners don't lose the requirement. |
| T-03-09 | Tampering | SSRF via `base_url` config | accept | `base_url` comes from local TOML file loaded by pydantic-settings (Phase 1). Not user input at runtime. httpx has `follow_redirects=False` by default on `stream()`. Single-user local tool. |
</threat_model>
