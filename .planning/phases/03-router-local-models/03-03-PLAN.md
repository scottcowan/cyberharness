<objective>
Implement the connectivity probe: a background asyncio task that TCP-probes `config.probe.probe_host` on a fixed interval, debounces state transitions with an N-of-M gate (`confirm_count=2`), and publishes `ConnectedEvent` / `DisconnectedEvent` to the Phase 1 EventBus on debounced transitions only.

Purpose: ROUT-01 — the harness needs a reliable, non-flapping connectivity signal that downstream subscribers (Phase 5 TUI status bar, Phase 6 QueueMgr, and Router's `is_connected` reader per D-15) can trust.     - `strategies.py`: `async def tcp_probe(host: str, port: int, timeout: float) -> bool`. Uses `asyncio.wait_for(asyncio.open_connection(host, port), timeout=timeout)`. Returns True on connect, False on `asyncio.TimeoutError` or `OSError`. `finally` block closes the writer if obtained: `writer.close()` then wrapped `try/except` around `await writer.wait_closed()` (Pitfall 6 — RST may hang or raise). No other exception handling.
    - `probe.py`: `@dataclass class ConnectedEvent: timestamp: float`, `@dataclass class DisconnectedEvent: timestamp: float`.
    - `Probe(config, bus, *, initial: bool = False, probe_fn=None)`. Reads `config.probe.probe_host`, `config.probe.interval_seconds`, and `getattr(config.probe, "confirm_count", 2)`. Stores `self.is_connected = initial` (default False — discretion documented in D-15 note and RESEARCH Open Question 2). `probe_fn` optional injection point for tests (defaults to module-level `tcp_probe`).
    - `_observe(ok: bool) -> bool | None`: N-of-M state machine per Pattern 3. Returns `True` on connected transition, `False` on disconnected transition, `None` on no transition. When `ok`, increment success_count, reset failure_count; when `not self.is_connected and success_count >= confirm`, flip state to True and return True. Symmetric for failures.
    - `async _check_once() -> bool`: call `self._probe_fn(self.host, 443, timeout=3.0)`. If False, call `self._probe_fn(self.host, 80, timeout=3.0)` (D-12 fallback; single retry on 443 failure — includes timeout AND connection refused, since we cannot distinguish without inspecting the exception type inside tcp_probe, and RESEARCH Pattern 2 notes the tradeoff — accept the extra port-80 attempt on connection-refused as low-cost).
    - `async run() -> None`: infinite `while True` loop. Each iteration: `ok = await self._check_once()`; `transition = self._observe(ok)`; if `transition is True`, `log.info("probe: connected")` then `await self.bus.publish(ConnectedEvent(timestamp=time.time()))`; if `transition is False`, `log.info("probe: disconnected")` then `await self.bus.publish(DisconnectedEvent(timestamp=time.time()))`. Wrap the check-and-publish in try/except so a single failure doesn't kill the loop, but let `asyncio.CancelledError` propagate. `await asyncio.sleep(self.interval)` at the end of every iteration (transition or not).
    - `probe/__init__.py` exports `Probe`, `ConnectedEvent`, `DisconnectedEvent`, `tcp_probe`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| harness process → probe host (default 1.1.1.1:443) | Outbound TCP only; no payload sent. Signal is binary (connect OK / not OK). No auth, no data on the wire beyond TCP handshake. |
| Probe → EventBus subscribers | Events are internal dataclasses; no external consumer in Phase 3. QueueMgr (Phase 6) and TUI (Phase 5) subscribe later. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Denial of Service | Loop-blocking probe | mitigate | `asyncio.open_connection` + `asyncio.wait_for` — never `socket.create_connection` (Pitfall 1). No sync I/O. Cancellation propagates cleanly. |
| T-03-11 | Denial of Service | Connectivity-flap thrash on marginal networks | mitigate | N-of-M debounce with `confirm_count=2` per D-14. Publish only on confirmed transition; downstream subscribers never see raw samples. |
| T-03-12 | Denial of Service | `writer.wait_closed()` hanging on RST | mitigate | Inner try/except around `wait_closed()` inside `finally` (Pitfall 6). Probe result already known before cleanup. |
| T-03-13 | Information Disclosure | Probe host reveals harness presence to a third-party DNS/IP | accept | Default `1.1.1.1` is Cloudflare DNS — anonymous, high-volume, no unique identification signal in a bare TCP connect. Operator can override via `config.probe.probe_host`. |
| T-03-14 | Tampering | Racing state between probe writes to `is_connected` and Router reads (D-15) | accept | Single asyncio event loop — no thread races. Router reads at `route()` entry and commits (Pitfall 8, D-15). Bool assignment is atomic in CPython. |
</threat_model>
